# Product

## Register

product

## Users

Agent 平台的开发者、平台管理员和业务使用者。他们需要在一个可信的工作台里创建智能体、选择智能体进行问答沟通，并查看具体智能体的运行状态、调用链路和监控信号。

## Product Purpose

这个 Web 应用是智能体平台的用户界面。它把智能体创建、智能体选择、对话沟通和监控观察放在同一个产品路径里，让用户能从配置到验证再到运行观察完成闭环。成功状态是：用户能快速理解平台状态，创建或选择一个智能体，开始对话，并在需要时看到这个智能体是否健康、可靠、可追踪。

## Brand Personality

务实克制、技术可信、执行导向。品牌参考 Agno 的 Agent 平台表达：强调高性能运行时、统一控制面、私有安全部署和可观察能力，但界面本身应服务任务，不做营销化炫技。

## Anti-references

不要做成泛 SaaS 营销页、装饰性 AI 炫光页面、只有聊天框但没有平台结构的 Demo、或过度拟人化的聊天产品。避免把监控和评估能力藏成二级细节；它们是智能体平台可信度的一部分。

## Design Principles

- 平台先于单点功能：创建、选择、对话和监控应被设计成一个连续工作流。
- 可信度可见：健康、延迟、任务状态、错误和监控信号要比装饰更优先。
- 中文优先，技术名词保留英文：用户可见文案默认中文，Agent、API、Trace、Monitor 等技术词按行业习惯保留。
- 低耦合模板感：界面可以表达智能体平台方向，但不要写死某个行业业务假设。
- 克制但不空：产品 UI 保持清晰、密度合理，重点状态和主操作用明确层级承载。

## Accessibility & Inclusion

默认按 WCAG AA 作为产品 UI 基线：正文对比度至少 4.5:1，键盘可访问，焦点状态可见，重要状态不只依赖颜色表达，并尊重 `prefers-reduced-motion`。动效只用于状态反馈、加载、切换和注意力引导。
